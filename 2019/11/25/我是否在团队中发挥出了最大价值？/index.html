我是否在团队中发挥出了最大价值？

发表于 2019-11-25 | 分类于 总结 | | 阅读次数:

如何提升技术能力？

如何提升业务能力?

如何提升项目（个人）管理能力?

  一眼看到这三个问题，对于职场上几乎所有人来说，都是提升自己的要点。而目前对于我来说，我认为我相对更欠缺的是个人的管理能力以及其他人的一些引领，因为这能约束好我去提高技术和业务能力。

一.首先，身在职场，那么说说我现在是如何规划自己提升技术能力的：

1.场景：目前大概就两种场景能够让我去提升自身技术

  ①工作时间：通过每个Sprint给出的需求，然后去做这个业务，这需要去理解熟悉原先的代码、设计自己本次的业务代码，还有一些当前sprint需要用到且不熟悉的技术。这些业务实现的完成大多依靠自身技术沉淀，仅有一些需要去学习才能完成的，但这部分带来的技术提升简直微乎其微，况且在做业务期间学习技术的时间会耽误整个需求完成的周期，所以应当花费时间当非工作时间当中。
  ②非工作时间：周二 ~ 周六：按照目前我对自己的计划，每天8小时下班延后2小时作为自己的学习时间。学习时间分配是：如果业务没有做完且很紧急那么2个小时都做业务，如果业务不是很紧急便拿出一个小时学习技术，如果没有业务则全部拿来学习。
  ③双休日：在这之前，双休日我主要是以’休’为主。现在的话我想给自己安排一些时间去做一些技术提升。目前的话是每天给出2小时时间。

2.方式：在学校时，我系统的学习一门技术主要是通过书、视频。也会碎片化的进行学习如看博客及与其他同学交流。

  ①书：我个人认为我是比较喜欢通过以书的方式来学习技术的，在校期间看了应该有接近或者超过25本技术书籍，没看完的更是数不胜数，但是有时候看完总感觉一带而过的感觉，我认为主要原因是在大多数情况下看书时我只是去摘录重点而非进行阶段性总结并写博客。所以，后面在以书为载体去学习时，我应该多做总结，撰写博客。这样便可梳理思路概述学到的内容，提升理解，真正将知识纳入囊中。
  ②视频：视频也是系统化学习的一种方式，书更重理论，而视频更重实操，更能够在短时间掌握一门技术，所以我认为视频是入门，而好的书可以做到技术升华。而最近犹豫很久应不应该系统学习Winform时刚好在做项目时碰壁，所以就打算通过视频恶补一番。

3.结合：所以周二~周六我会通过制定的短期学习方向及目标去看书或者视频，再拿出部分时间去做算法题。周末的话就拓展自己的技术广度，看更多其他方向的微技术书籍做到了解以及每天做一道算法题。

  很多时候，学习技术的困惑在于，什么时候我能够用到它？这是我不舍于花时间学习技术的部分原因。而现在，我不应该再犹豫了，规划自己中长期在业务中需要用到的技术，就去学它吧。

二.那么，如何提升业务能力呢？

我总结了三点，就是“看问写”。

1.多看：

  ①各种文档：不管是在学习业务还是在做业务期间，多翻前辈们写下的文档，总能提取很多与业务相关信息供为己用，做到快速了解产生自己对相关业务的理解与能够把这个业务做下去的思路，这样便大大提升了工作效率。
  ②各种书籍:为什么要多看书呢？我认为看书不仅可以提高理解能力还能够写出更好的文档，而提高了理解能力在看文档能做到更快的吸收、拿到需求时更快的分析，写出更好的文档能够在以后分配到相关业务的时候更快上手以及让其他人在做与此业务相关的需求时也能够更好的了解。(目前看书我会每天给自己大于20分钟的时间去阅读各种书籍)

2.问：

  ①除了看，另一种吸收业务知识的方式就是问，有时候理解不了的业务问题甚至其他各种问题去问前辈们大部分时候都能得到很好的解决方案。

3.写好：

  ①需求分析文档：每次拿到需求的第一件事就是分析需求产生相应需求文档，而好的需求文档能够让整个项目做到事倍功半。而如何写好呢？目前我的话就是通过看和问以及不断总结去改善上次需求中产生的短板，避免在下次做新需求时重蹈覆辙。
  ②项目总结文档：总结我觉得是非常非常重要的，因为我们做的事情总是会重复的，为了避免重复犯错以及总结规律提升做事效率，我们就要在每次项目结束时在脑海里复盘整个项目过程提取出问题产出总结文档。

三.从7月走出校园到现在11月在公司工作四个月，我觉得我还正处在摸索阶段，不管生活、工作各个方面我都还在寻找更好的方式来提升自己，那么我应该如何提升自己的项目(个人)管理能力以便快速提升我的技术能力以及业务能力而不是做一个原地踏步的技术人员呢？

1.定个人计划：

  ①短期：我一般会定近一个月的学习计划去学习相关技术，然后通过每天分配的非工作时间去学习它。
  ②中长期：在正式工作一年后成为中级开发工程师。
